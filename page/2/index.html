可以说，吕振中牧师的一生，特别是后半生是这个原则的完美彰显。他专注翻译圣经一件事，把它做好了；因此，祝福了全世界的华人基督徒，也成了众人的榜样。

怀念Tom：追思礼拜有感

发表于 2017-08-07 | 分类于 生活日记 | | 阅读次数

上周二神学院IT部门的弟兄Tom 发现死在自己的车中，明显是因为吞食过量的药物自杀身亡。

这个消息在神学院中引发强烈的震惊，所有的回应都是：Oh, No. 没有人会想到，Tom弟兄会自杀。他平时在人们眼中，是那么友善、热情，乐于助人；别人看来，他的脸上总是带着微笑。他是个好同工，技术能力强，非常善于解决问题。
我跟他接触不多，去IT寻求帮助，也就是寻求帮助，也没有时间聊多少。每次在神学院碰到他，他总是微笑，很亲切的弟兄。

没有人知道他经历了什么挑战和难处，看起来他一切都Ok。就在他自杀前一周的周末，他还来神学院上班，大家都看到他了，没有感觉有什么异样。他的微笑和他的自杀是那么的不协调，让人无法理解。国际学生部的负责人Sarah说：It doesn’t make any sense. 是啊，How？

好的工具激发灵感：选择合适的blog平台

发表于 2017-08-06 | 分类于 工具控 | | 阅读次数

精彩摘录

我希望我写的blog可以在其它地方被管理和使用，也就是说blog不能成为一个自我封闭的体系，它需要能与其他的文档进行交流、互动，再使用（比如出版成书）。leanote作为一个笔记系统具备这样的潜质，但问题在于它的笔记管理系统是否是我想要的工具。我是否要把我的知识管理系统迁移到leanote，不然其实没有意义！目前的答案是否定的，它的开放性还不够。但开源给了它潜力，同时也可能成为软肋。我现在觉得以nvALT为代表的(markdown) txt管理系统，是最快捷的。加上The Archive的开发，我觉得用Zettle来管理我的知识，特别是学术研究非常有效。
我已经非常习惯用markdown写作，它把内容和格式分离，非常灵活；加上pandoc工具，我可以用它进行纯文本写作，并且编译出好看的论文文档。我的整个神学硕士就是用markdown写成的。

文章正文
就如我在另外一篇文章继续blog，继续成长提到的，我换了好多个博客，其中一个原因是：我不断的学习如何使用最新的技术更有效的学习和分享我所学习的。好多的博客系统作为工具，不再更新他们的功能。这也能理解，因为当一个东西积累了一定的用户之后，用户的习惯就是关键的决定因素了。我就听说这样的故事，说可口可乐要更新他们的新配方，结果，招来消费者的大量投诉、抗议，最后只好又回到原来的配方上，只是在包装上做出改变。当用户习惯了，用户的口味就是一切。
